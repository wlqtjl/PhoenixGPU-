### 🔥 新特性：GPU 热迁移（Live Migration Alpha）

在不中断训练的情况下，将运行中的 GPU 任务从一个节点迁移到另一个节点。

```bash
# 触发热迁移
kubectl phoenixgpu migrate \
  --job research/llm-pretrain \
  --target gpu-node-02

# 或通过 API
curl -X POST http://phoenixgpu-api:8090/api/v1/jobs/research/llm-pretrain/migrate \
  -d '{"targetNode":"gpu-node-02"}'
```

**迁移流程**（总时间 < 120s，冻结窗口目标 < 5s）：
1. `PreDump` — 非侵入式预写脏页（进程不暂停）
2. `Dump` — CRIU 全量快照（**冻结窗口开始**）
3. `Transfer` — 节点间 rsync 传输快照
4. `Restore` — 目标节点 CRIU 恢复（**冻结窗口结束**）

**当前限制（Alpha）**：
- 仅支持同 Kubernetes 集群内节点间迁移
- 需要节点间 SSH 连通性（rsync 传输）
- `execOnNode` 通过 K8s SPDY exec API 实现（无 kubectl 依赖）
- 恢复失败时自动 SIGCONT 解冻源进程

### 💾 计费系统：TimescaleDB 真实存储

- `usage_records` 超表（按月分区）
- 90 天数据保留策略 + 7 天后自动压缩（~20×）
- 连续聚合视图：`daily_dept_summary`（WebUI 查询性能 10× 提升）
- 安装命令：`psql $DSN -f deploy/db/001_initial_schema.sql`

### 🚀 升级方式

```bash
helm repo update
helm upgrade phoenixgpu phoenixgpu/phoenixgpu \
  --version 0.2.0-alpha \
  --namespace phoenixgpu-system \
  --set migration.enabled=true
```
